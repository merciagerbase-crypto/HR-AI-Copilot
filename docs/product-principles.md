# Product principles

## 1. AI assists. Humans decide.
The products may summarize facts, structure a case, suggest questions and surface risks. Managers, HR and designated decision-makers remain accountable for all employment decisions.

## 2. The product should be specific.
Each workflow starts from a concrete job to be done. Generic chat is not the product. A repeatable decision-support flow is the product.

## 3. Escalation should be explicit.
Sensitive cases receive a visible handoff recommendation to HR, Legal, Compliance, Employee Relations or Security. The product does not hide uncertainty.

## 4. Data minimization is a feature.
A production version should request only the facts needed for the workflow. It should avoid protected characteristics, diagnosis data and unnecessary free text.

## 5. Outputs need traceability.
A production version should retain source references, the user-provided inputs, model version, prompts, output, reviewer decision and final action status.
