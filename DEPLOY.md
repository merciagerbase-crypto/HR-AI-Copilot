# Como deixar os dois produtos acessíveis para a entrevista

## Opção mais rápida: GitHub Pages

1. Crie um repositório no GitHub com o nome `hr-ai-product-portfolio`.
2. Descompacte o arquivo `.zip` e envie todos os arquivos da pasta para a raiz do repositório.
3. Abra **Settings > Pages**.
4. Em **Build and deployment**, selecione **Deploy from a branch**.
5. Selecione a branch `main` e a pasta `/root`.
6. Salve. O GitHub mostrará uma URL pública em poucos minutos.

A URL principal abre a página de portfólio. Os dois produtos ficam em:

- `/people-manager-copilot/`
- `/workforce-planning-copilot/`

## Nome e descrição sugeridos para o repositório

**Nome**: `hr-ai-product-portfolio`

**Descrição**: `Interactive HR AI product prototypes: manager decision support and workforce planning.`

## O que dizer ao entrevistador

> I built two small product prototypes to show how I think about AI in HR. One supports managers preparing for sensitive people conversations. The other helps HR, Finance and business leaders prioritize workforce decisions. Both are designed with human review, clear decision boundaries and auditable outputs.

## Antes de enviar

- Troque a descrição do repositório, se quiser, pelo seu nome ou LinkedIn.
- Clique nos dois botões de geração para demonstrar os fluxos.
- Não diga que os produtos usam dados reais. Todo o conteúdo foi construído com dados sintéticos.
